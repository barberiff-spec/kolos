# Рефакторинг Kolos — сводка изменений

Важно: код правился без доступа к интернету в моей песочнице, поэтому
проект не собирался и не запускался (`npm install` / `pip install`
недоступны). Изменения проверены вручную построчно и синтаксис backend
проверен через `python -m py_compile`. Перед деплоем прогони обычный
локальный билд/тесты.

## 1. iOS Safari / WebKit — основная причина багов рендеринга

Причина в трёх независимых, но связанных вещах:

**a) `100vh` вместо `100dvh`**
На iOS Safari `100vh` не учитывает скрывающуюся/появляющуюся адресную
строку — контейнер либо обрезается, либо появляется "прыжок" при скролле.
Заменено на `100dvh` (с `100vh`-фолбэком в `globals.css` для очень старых
браузеров) в:
- `frontend/src/app/layout.tsx`
- `frontend/src/app/auth/login/page.tsx`
- `frontend/src/app/learn/[courseId]/page.tsx`
- `frontend/src/app/learn/[courseId]/learn-content.tsx`

**b) `backdrop-blur` на `sticky`-элементе внутри вложенного скролл-контейнера**
Это задокументированный баг WebKit: `position: sticky` + `backdrop-filter`
внутри `overflow-y-auto`-контейнера (не document body) не перерисовывается
при скролле — блюр залипает или пропадает. Убрал `backdrop-blur` со
sticky-хедера урока (`learn-content.tsx`), заменил на сплошной фон — той же
логике уже следовал `.glass` в `globals.css` для мобильных (комментарий
"No backdrop-blur on mobile — Safari freezes on heavy blur layers"), теперь
она применена последовательно.

**c) `position: fixed` фон без промоушена в GPU-слой**
`PremiumBackground` — полноэкранный fixed-слой с градиентами — на Safari
мог перерисовываться (мигать/тормозить) при каждом скролле страницы.
Добавлен `transform: translateZ(0)` + `will-change: transform`, чтобы
браузер держал его в отдельном composite-слое.

**d) Momentum-скролл**
Добавлен класс `.ios-scroll` (`-webkit-overflow-scrolling: touch`) на
скроллящиеся панели урока (сайдбар + контент) — без него вложенный скролл
на iOS иногда "залипает" на середине жеста.

**e) Safe area**
`viewport.viewportFit: "cover"` в `layout.tsx` — контент корректно
уходит под чёлку/home indicator вместо белой/чёрной полосы.

## 2. Backend

- `backend/app/main.py`: `@app.on_event("startup")` — устаревший API,
  заменён на `lifespan` context manager (текущий стандарт FastAPI).
- `backend/app/core/config.py`: список продакшн-доменов для CORS вынесен
  из тела валидатора в именованную константу `KNOWN_PRODUCTION_ORIGINS` —
  было "магическим" инлайн-списком, теперь видно и легко расширять.

## Не тронуто (сознательно)

- `backend/app/models/__init__.py` (245 строк) и `services/seed.py`
  (421 строка) — оба разрослись и по-хорошему стоит разбить на модули, но
  без возможности прогнать импорты/тесты в песочнице делать это вслепую
  рискованно. Отдельно скажи, если хочешь, чтобы я сделал это следующим
  шагом — сделаю аккуратно, с `__init__.py`, реэкспортирующим всё старое,
  чтобы точно не сломать импорты в остальном коде.
- Остальной код (API-клиент, auth store, deps.py и большинство эндпоинтов)
  уже в хорошем состоянии — без `console.log`, без `any`, без голых
  `except:` — трогать не стал, чтобы не вносить риск без пользы.
